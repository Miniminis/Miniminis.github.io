Node.js / 기본 개념 01 - node 특징

2019-10-14

#nodejs

지난 연차 프로젝트에서 간단하게 노드 서버를 구축해보고 나니, 노드에 대해서 더 공부하고 알아가고 싶어졌다. 그간 계속 자바+스프링 프로젝트를 해오다보니 노드가 훨씬 가볍고 빠르게 느껴졌다. (물론 부트를 써 본 것은 또 다른 새로운 경험 중 하나였다. ) 그래서 조현영 개발자님의 Node.js 교과서를 공부하면서 조금씩 이곳에 기록을 남기고자 한다. 훗날 이전에 공부했던 개념과 내용들이 헷갈릴때, 언제든지 다시 와서 펼쳐볼 수 있도록! + 노드를 공부하는 나와 같은 사람들에게도 도움이 될 수 있도록.

자바스크립트 런타임

공식 홈페이지 소개글에 따르면, 노드는 이러한 언어이다.

Node.jsⓇ는 크롬 V8 자바스크립트 엔진으로 빌드된 자바스크립트 런타임입니다. Node.js는 이벤트 기반, 논블로킹 I/O 모델을 사용해 가볍고 효율적입니다. Node.js의 패키지 생태계인 npm은 세계에서 가장 큰 오픈 소스 라이브러리 생태계이기도 합니다.

불과 10여년 전까지만 하더라도 자바스크립트는 실행속도의 문제 때문에 브라우저가 아닌 다른 곳에서는 실행하기가 힘들었다고 한다. 그러나 구글에서 V8엔진을 사용해서 크롬을 출시한 뒤부터는 자바스크립트의 실행속도가 몰라보게 발전했다고 한다. 이러한 환경에서 라이언 달(Ryan Dahl)에 의해 고안된 것이 바로 Node.js 이다.

이벤트 기반 & 논블로킹 I/O

노드가 실행되는 원리는 다음과 같다. 스크립트 언어를 쓰여진 순서대로 쭉 읽어내려가다가 함수가 호출되거나 이벤트가 발생된다. 그러면 해당 함수는 바로 호출스택으로 이동하여 쌓이게 된다. 이벤트 루프는 바로 이 호출 스택에 쌓인 함수들을 관리하고 함수들의 실행 순서를 결정하는 역할을 한다. 호출스택에 있는 이벤트 혹은 함수들이 실행되면 미리 노드 서버에 정의된 이벤트리스너에 의해서 그 실행 결과가 반환된다. 이러한 반환 결과 혹은 이벤트 리스너의 콜백함수들은 테스트 큐에 줄지어 대기하게 된다. 호출 스택이 모두 비워져야만 이벤트 루프가 테스크 큐에 대기하던 콜백함수 또는 결과들을 호출스택으로 가져와 실행시킨다. 테스크 큐도 비워지게되면, 이벤트 루프는 다음 이벤트가 호출될 때까지 계속 대기하고 있는다. 노드 서버가 종료될 때까지 이 작업들이 반복되기에 이벤트 루프라고 한다.

한편, 이벤트 루프는 시간이 오래 걸리는 파일 입출력이라던지, 일정시간 뒤에 실행되는 setTimeout() 과 같은 함수들이 호출될 경우에는 바로 백그라운드로 보내 그곳에서 실행이 되도록 하며 바로 다음 동작을 실행한다. 백그라운드에서 완료된 작업 결과는 바로 테스크 큐에 줄서게 된다. 싱글쓰레드 언어인 자바스크립트가 마치 멀티쓰레드처럼 두 가지 일을 동시에 하는 효과를 내는 것이다. 이처럼 오래 걸리는 작업을 백그라운드에 미뤄두고 바로 다음작업을 진행한 뒤, 오래걸리는 작업의 결과가 나오면 표현해주는 것이 바로 논블로킹의 개념이다.

정리

  • 이벤트 : 클릭 or 네트워크 요청 사용자가 웹 페이지 상에서 발생시킬 수 있는 사용자의 움직임?

  • 이벤트 + 이벤트 리스너 (콜백함수): 짝꿍

  • 이벤트 루프 : 호출된 콜백함수의 실행순서 결정하는 역할. 노드 종료까지 이벤트 처리를 위한 작업을 반복 –> 루프!

  • 태스크 큐 : 이벤트 발생 후 호출되어야 할 콜백 함수들이 기다리는 공간

  • 백그라운드 : 타이머나 I/O 작업 콜백처럼 시간이 오래걸리는 작업 또는 이벤트 리스너들이 대기하는 곳

  • 블로킹 : 작업이 호출되는 순서대로 시작-끝. longRunningTask()

  • 논블로킹: 오래 걸리는 작업을 백그라운드에 미뤄놓고 다음 작업부터 처리하는 것. setTimeout(longRunningTask, 0);

싱글 스레드

자바 스크립트는 싱글스레드 언어이며 자바스크립트를 기반으로 하는 Node.js 역시 싱글쓰레드이다.

  • 스레드 : 쉽게 말하자면 컴퓨터가 작업을 처리하는 일손이다. 식당에 있는 종업원의 개념으로 이해했다.
  • 멀티스레드 : 여러개의 스레드가 일을 나눠 처리함. 많은 일손이 각각 한가지의 일을 담당해서 처리.
  • 싱글스레드 : 하나의 스레드가 한번에 한 가지의 일을 처리함.
  • 프로세스 : 운영체제에서 할당하는 작업의 단위. 프로세스 간에는 메모리 등 자원 공유를 하지 않음.
  • 스레드 : 프로세스 내에서 실행되는 흐름의 단위. 하나의 프로세스는 여러개의 스레드를 가짐. 스레드들 간에는 부모 프로세스의 자원(메모리)을 공유함.
