Hexo 프레임워크 이용하여 블로그 만들기

2019-09-16

필요

책읽는 데에는 한없이 게으르지만, 글을 통해서 사람들과 소통하는 것을 좋아했다. 다른 사람들을 글을 읽으며 그들의 세계에서 잠시 헤메고 그들과 생각을 공유하고 그런 과정들이 재미있었다. 그렇게 바빴던 케냐에서도 그 순간을 잊지않고 기록하며 아프리카와 케냐를 모르는 다른 사람들에게 조금이나마 그곳을 전하기 위해 글을 썼었다. 나중엔 게을러지고 너무 바빠져 역시 포스팅이 미진했지만.

블로그를 직접 만들어야겠다고 생각한 것은 개발이라는 새로운 세계와 만나고 나서부터였다. 개발은 그 범위가 너무나도 광활하고 깊어서 새로운 개념이나 에러와 마주한 순간마다 북마크를 해놓는 것만으로는 결코 충분하지 않았다.

개발 세계에 조금 더 가까이 다가갈수록 흥미롭고 매력적이라고 느꼈지만, 뭔가 미로에서 길을 잃은 듯한 답답함이 있었다. 모든 것을 기억하고 공부할 수는 없었고, 다만 다음번에 같은 문제나 개념을 만났을 때, 길을 찾아가는 법 정도는 알아야겠다고 생각했다.

요약 : 조만간 크롬 북마크 폴더들이 터질 것 같아서 시작


참고 사이트

hexo 블로그를 만들면서 내가 참고했던 사이트들을 아래에 소개한다. 같은 내용의 반복일 것 같아서 따로 포스팅은 하지 않겠다. 다만, 블로그에 나와있던 내용 이외에 개인적으로 맞닥뜨린 어려움들에 대해서만 간단히 기술하고자 한다.


Github Page와 Hexo를 통해 30분만에 기술 블로그 만들기

정말 친절하게 hexo 설치부터 테마적용, git 연동까지 자세하게 포스팅을 적어주셨다. 이 포스팅을 처음부터 따라하면 정말로 30분도 안걸려서 블로그를 만들 수 있다.

다만 내가 헷갈렸던 부분은

  1. git repository를 만들고 + clone 한 뒤에 해당 repository에서 hexo init 을 실행하는 것인지,
  2. 아니면 다른 외부 폴더에서 hexo init을 실행하는 것인지
    였다.

이런 부분 때문에 처음에는 외부 폴더에서 hexo init 을 실행하여 약간의 삽질 과정을 겼었다.


Hexo로 블로그 만들기

위의 블로그에서 헷갈렸던 부분을 이 블로그를 보면서 해소했다.
이 분은 (본인이 삽질을 겪으신 것인지는 몰라도) 각 명령어를 실행하는 위치를 명확하게 말씀해주시며 처음 hexo를 이용하는 사람들이 헷갈리지 않도록 아주 쉽게 설명해주시고 있다.

이 분 덕분에 테마도 잘 적용하고 성공적으로 github page 도 만들었다 룰루!


hexo 공식 사이트

hexo 공식 사이트는 정석으로 알려주지만, 뭔가 각 명령어의 구체적인 적용된 모습을 확인할 수 없어서 아쉽다. 하지만 기본 구조, 개념, 설명 등은 이곳이 가장 정확하므로 이 사이트와 실제로 적용된 모습을 설명하고 있는 개인 블로그, 이 두개를 같이 번갈아가면서 보는 것이 도움이 될 듯.


git 마크다운방법 설명 문서

markdown 방법이 익숙하지 않은 사람들은 github에서 제공해주는 문서를 참고하면 쉽게 따라할 수 있을듯. html이나 css 등에 익숙한 사람이라면 더 쉽게 익힐 수 있는 것 같다.
예제도 같이 나와있어서 따라하기 정말 좋음.


내가 겪은 어려움

  • 테마 커스텀

현재 이 블로그에서 사용하고 있는 테마는 anodyne 으로 해당 테마의 깃 사이트에 가면 README 에 엄청 자세하게 테마 적용하는 법부터 커스텀까지 설명해놓았다. 나는 다만, style.styl 와 같이 .styl 확장자를 가진 파일을 처음봤고 뭔가 css랑 비슷한 것 같은데 오묘하게 문법이 달랐다.
그래도 css 할 줄 아는 분이면 테마에 적용된 스타일 파일을 찾아서 잘 커스텀 하실 수 있을 듯.

  • Git 연동

위의 첫번째 블로그를 따라하고 있었는데 이상하게 모든 설정을 다 마치고 git으로 커밋을 하면 index.html 파일이 사라져 404 에러가 떴었다. 이 문제는 두 번째 블로그를 보면서 해결했다.

오늘의 이 글이 도움이 되었기를 바라며, 즐거운 포스팅 하세요~!
