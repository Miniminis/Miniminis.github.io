#nodejs

Node.js

  • 2009년 Ryan Dahl

  • 웹 프로그램(브라우저)에서 어플리케이션 개발이 가능하도록 만들어 놓은 프레임워크

  • 장점 :

    • 자바스크립트 기반 언어 : 클라이언트와 서버에서 사용하는 언어가 같다
    • 싱글 쓰레드, 네트워크 어플리케이션에 적합
    • 비동기통신을 통한 입출력/이벤트 기반**
    • 모듈과 패키지(npm 설치관리자)를 사용하면서 서버 프로그램 구성
    • 이미 만들어진 모듈이 많아서 개발하기에 편리

  • 권장분야 :

    • 실시간 소셜 네트워크 서비스
    • 데이터 중심의 서비스
    • IoT 기기 연동

  • 다운로드 :

    • 링크
    • 안전하게 Recommended For Most Users 버전을 설치하는 것이 권장됨

개발 환경 설정

  • REPL : 콘솔 기반 실행환경
  • IDE :
    • Eclipse, Webstorm
  • Editor :
    • Bracket
    • Visual Studio code(추천)

모듈 사용하기

  • 클래스: interface

  • 메소드

  • 이벤트

  • 모듈함수

  • 모듈 로딩

    • require(‘모듈이름’)
    • 절대 경로 or 상대경로

  • 모듈 종류

    • 기본 모듈 : 이미 컴파일 된 상태로 라이브러리 디렉토리에 위치(Node.js 라이브러리 디렉토리)
    • 확장 모듈 : npm 이용해서 별도로 설치해야함(node_modules 이름의 폴더)

  • 모듈 로딩 : 객체 생성

    • 클래스
    • 모듈 로딩과 객체 생성
    • Readline 의 경우 클래스는 interface

  • 객체 생성

  • 메소드 사용

    • 모듈 로딩, 객체 생성 후 메소드를 사용

  • 이벤트

  • 모듈 함수
    • 객체 생성 없이 모듈에 직접 사용
