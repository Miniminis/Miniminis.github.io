스프링 부트, 왜 써야할까?

  1. 초기 셋업에 드는 시간이 기존 스프링 프로젝트에 비해 훨씬 절약됨!
  2. 반복되는 설정 사항들이 이미 부트 내부에 있어서 특별히 설정해야할 부분만 선택적으로 추가하면 됨
  3. 톰캣이나 제티 등 웹 어플리케이션 서버를 내장하고 있어서 프로젝트를 띄우는 시간이 훨씬 적다.
  4. 의존 라이브러리를 자동으로 완성 : pom.xml 에서 일일히 지정하지 않고 부트가 권장 버전을 관리함.

한마디로, 귀찮은 설정 작업과 프로젝트 시작에 드는 시간이 훨씬 절약된다.
