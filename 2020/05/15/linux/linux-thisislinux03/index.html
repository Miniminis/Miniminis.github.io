이것이 리눅스다(2020) 03 / CentOS8 Linux 설치 - ServerA

2020-05-15

‘이것이 리눅스다 2020년 개정판’을 공부하고 있다. 나와 같이 블로그나 글을 통해 공부하는 것이 더 편한 사람들을 위해 공부의 과정을 간단하게나마 글로 남기고자 한다. 책은 [링크]를 통해 구입할 수 있고, 강의는 [링크]에서 확인할 수 있다. 한빛에서 네이버 카페[링크]도 운영하고 있어서 정말 편하게 공부하고 있다. (열심히 공부해서 성공하겠습니다…! 감사합니다…!)

이번 포스팅에서는 ServerA의 설치과정을 다룰 것인데, 강의를 들으면서 급하게 필기한 내용들이라 자세한 내용은 위의 링크를 통해 강의를 직접 참고하는 것을 추천한다. 이 글은 놓친 텍스트나 나중에 흐름을 정리하고자 할때 간단히 참고해주시면 될 듯.


1. 설치 및 설치파일 체크

1) DVD ISO 다운로드

  1. CentOS8 1905 버전의 DVD ISO 파일을 다운로드
  2. 체크섬 프로그램을 다운받아서 해당 ISO 파일이 문제가 없는지 확인하도록 한다.
  3. 나는 위의 과정을 생략하고 그냥 서버 설치에 들어가긴 했다. 하지만 설치과정에서 문제없이 한번에 깔끔하게 끝낼 수 있도록 파일 이상여부를 체크하는 것이 좋겠다.


2) 간단한 환경설정 및 계정등록

  1. 설치 후 > vmware player 실행 > serverA 에 다운로드 받은 ISO 파일 업로드 후 > player start
  2. enter : 삽입한 dvd 검사 > 문제 없다면 설치 진행 시작 > 다운로드 완료
  3. 언어 선택 : 한국어
  4. 시간대 선택 : 서울
  5. 네트워크 & 호스트 이름 선택 : 이더넷 ens160 > 켬
  6. 소프트웨어 선택 :
    1. 서버-GUI : 기능이 많지 않고 사용에 불편
    2. 워크스테이션 선택
  7. 설치 목적지 :
    1. 로컬 표준 디스크 선택 > 저장소 구성 : custom
      1. swap : 4G
      2. / (root) : 나머지 (76G)
  8. 설치 시작 클릭
  9. 설치 되는 중 다음을 진행한다.
    1. root 암호 : thisisadmin
    2. user creation :
      1. name : centos
      2. pw : thisiscentos
  10. 재부팅


2. 로그인

  1. 기본적인 세팅을 위해 root 로 로그인 (서버의 경우는 항상 root 로 로그인 하도록 함)
  2. 로그인 후 필요한 세팅 작업
    1. 화면사이즈
    2. 자동 화면잠금(끔)
    3. 계정 설정


3. 서버 설정

1) 자동업데이트 차단

  1. 상단 현재활동 > 터미널:

    1. 자동 업데이트 차단 command (강의를 듣는 동안 centos 1905 버전이 유지될 수 있도록)

      1. gsettings set org.gnome.sofeware download-updates false
      2. systemctl disable dnf-makecache.service
      3. systemctl disable dnf-makecache.timer

    2. 소프트웨어 업데이트 차단

      1. cd /etc/yum.repos.d/

      2. mkdir backup

      3. ls

      4. mv *.repo backup/ : backup folder 로 전부 이동

      5. gedit This.repo : 메모장

      6. 아래의 내용을 붙여넣기

      7. dnf clean all 로 변경된 repo file 적용

      8. exit


2) 네트워크 세팅

서버의 IP를 고정 IP로 바꾸는 작업

  1. cd /etc/sysconfig/network-scripts/

  2. ls :
    return ifcfg-ens160

  3. gedit ifcfg-ens160

  4. 바뀐 네트워크 설정 적용된 것을 확인하기 위해 재부팅

    1. nmcli connection down ens160 : 네트워크 내림
    2. nmcli connection up ens160 : 네트워크 다시 올림
    3. reboot : 재부팅
    4. ifconfig ens160 으로 바뀐 ip 주소 확인 (고정되었는지)


3) 편의성을 위해 보안 강화된 부분 끄기

  1. gedit /etc/sysconfig/selinux

  2. 저장 후 닫기


4) 한글설정

  1. 오른쪽 상단 전원
  2. 지역 및 언어
    1. 한국어 하나만 남기고 나머지 지우기
  3. 터미널 :
    1. shift + space : 한영 전환


5) 해상도 설정

  1. 터미널 : gedit /etc/grub.d/10_linux
    1. 169행 : “{args} vga=773”
    2. 저장 후 닫기
  2. grub2-mkconfig -o /boot/grub2/grub.cfg : 수정내용 적용
  3. done 나오면 완료
  4. 재부팅 후 확인


6) 방화벽 기능



7) gnome 소프트웨어 업데이트 차단

  1. 메뉴 > 프로그램표시 > 모두 : gnome software > 소프트웨어
  2. 상단 소프트웨어 > 업데이트 기본설정 > 자동업데이트 알림 끄기
  3. 닫기

9) 스냅숏 기능 설정

Workstation pro만 제공해주는 기능으로 player에서는 사용불가 ㅠㅠ
서버 설정의 과정이 만만치 않기에 일단 서버를 성공적으로 설치하고 설정을 완료한 뒤에는 스냅숏 기능으로 이 상태를 저장해놓는 것이 좋다.

  1. ServerA 열기
  2. VM > snapshot > snapshot manager
  3. take snapshot 클릭 :
    1. 이름은 마음대로. 강의에서는 설정완료로 하길래 따라서 함.
    2. 현재 세팅된 상태까지 저장된다.
  4. close
  5. 스냅숏이 제대로 적용 되었는지 테스트 :
    1. player 실행 > serverA booting
    2. root 계정으로 login
    3. 터미널
      1. rm -rf /boot : booting 과 관련된 디렉토리를 삭제함
      2. reboot 재부팅 명령어 먹지 않음
    4. workstation pro 열기
      1. snapshot > snapshot manager
      2. 설정완료 : goto click

이 과정에서 거짓말처럼 당황스러운 상황을 만났는데, grub rescue 모드에서 ls 명령어 말고는 아무것도 먹히지 않았었다. 결국 호스트 컴퓨터를 강제 종료하고 workstation pro 로 돌아가서 snapshot 으로 원복했다. 휴우…
