‘이것이 리눅스다 2020년 개정판’을 공부하고 있다. 나와 같이 블로그나 글을 통해 공부하는 것이 더 편한 사람들을 위해 공부의 과정을 간단하게나마 글로 남기고자 한다. 책은 [링크]를 통해 구입할 수 있고, 강의는 [링크]에서 확인할 수 있다. 한빛에서 네이버 카페[링크]도 운영하고 있어서 정말 편하게 공부하고 있다. (열심히 공부해서 성공하겠습니다…! 감사합니다…!)


네임서버 개념

  1. 네임서버 = DNS(Doname Name System) 서버
  2. 도메인의 이름을 ip 주소로 변환시켜주는 서버

네임서버 발전 과정

  1. 가장 초기의 네트워크 접속 방법은 ip 주소를 주소창에 직접 입력하는 것이었다.
    1. ex) 1.1.1.1
    2. 컴퓨터가 몇 대 없었기 때문에 사용자가 모두 외워서 직접입력
  2. 그러다가 서버의 개수가 조금씩 늘어났고 하나의 파일에 서버의 ip 주소와 별칭을 붙여서 매핑해놓는 방법으로 인터넷에 접속했다.
    1. hosts file
    2. URL - IP address mapping
  3. 서버가 기하급수적으로 늘어난 뒤에는 파일 하나에 모든 ip와 별칭을 매핑해놓는 것이 불가능해졌다. 이름을 전문적으로 매핑해주고 이를 해석해주는 서버가 필요하게 되었고, 네임서버가 발전하게 되었다.
    1. 이제 사용자는 ip 주소를 더이상 신경쓰지 않게 되었고
    2. url 만으로도 어디든지 접속할 수 있게 되었다.
    3. 프로세스
      1. 사용자가 url 을 입력
      2. 네임서버는 ip 주소를 반환
      3. 해당 ip 주소로 사용자는 원래의 목적지에 도달!
    4. 따라서 우리가 웹상에서 입력하는 주소는 사실상 네임서버로 전송되어 다시 ip반환을 통해 우리가 원래 가고자 했던 곳으로 이동하는셈.



/etc/hosts 파일 설정 확인 실습

하고자 하는 것 :

  • 사용자는 웹브라우저를 통해 특정 웹사이트에 도달하고 싶을 때마다 네임서버(도메인서버)에 해당 웹사이트의 ip 주소를 질의하여 받아온 뒤 접속한다.
  • 그럼 해당 네임서버에 웹사이트의 ip주소가 존재하지 않을 경우에는 어떻게 되는지 확인
  • 해당 네임서버에 다른 ip 주소가 매핑되어있으면 어떻게 되는지 확인
