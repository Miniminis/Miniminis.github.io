시간이 많이 없어서 틈틈히 짬나는대로 책을 읽고 정리하고 있습니다.


함수, 차곡차곡 쌓이는 스택

함수는 영어로 function 이라고 하죠. 단어 그대로 사용하면 무엇인가 유용하게 사용할 수 있는 기능입니다.

프로그래밍을 할때, 자주 사용되는 코드들을 함수로 정의해놓고 쓰다보면 불필요하게 반복되는 코드를 쓸 필요도 없어지고, 흐름도 좀 더 명확해지며 가독성도 좋아집니다.


맨 처음 유투브를 보며 이것저것 따라하면서 코딩을 할 때에는 함수가 정확히 어떤 원리로 작동되는지 알지 못했습니다. 그냥 필요한 기능들을 미리 정의해놓고 가져다가 쓰면 되는구나~ 정도였죠. 오늘은 함수가 호출될때 일어나는 일들을 살펴보면서 그 원리에 대해서 알아봅시다!


지역변수와 전역변수

전역변수는 함수 밖에 선언된 변수입니다. 그 범위가 넓죠.

지역변수는 함수 내에 선언된 변수입니다.
위의 실행결과를 보면, 함수 내에서는 전역변수의 값을 가져와 변경할 수가 없는 것으로 보입니다.

함수 내부에서 전역변수의 값을 변경하고자 할 때에는 두 가지 방법을 이용합니다.

1. global keyword 를 사용해서 local 매소드 내에서 전역변수를 변경

2. nonlocal keyword 이용해서 local 매소드 내에서 전역변수를 변경


인자전달방식에 따른 함수의 분류

함수는 인자 전달방식에 따라서 아래와 같이 분류될 수 있습니다.

  1. 값에 의한 전달
  2. 참조에 의한 전달
  3. 객체참조에 의한 전달

값에 의한 전달 (C언어)

함수 내에서 출력한 내용은 변했는데, 메인 함수에서 다시 확인해보니 변수 x 값은 그대로입니다. 이유는 변수 x를 전달한 것이 아니라, x의 값을 전달했기 때문!!

스택프레임 개념

함수가 호출될 때 메모리 내에서는 “스택프레임”이 생긴다. 함수 내에 정의되어있는 변수들이 메모리를 차지하는 공간이 생기고 함수가 호출되는 순서에 따라서 차곡차곡 위로 쌓인다 –> 스택구조로! 그래서 제일 나중에 호출된 함수가 가장 먼저 사라지고, 제일 먼저 호출된 함수는 가장 나중에 사라지게 된다.


참조에 의한 전달 (C언어)

(그 악명높은… 말로만 듣던…) 포인터 변수를 통해서 메모리주소값을 직접 넘겨서, 함수내에서 메인함수에 정의된 변수의 값을 변경하였습니다!!!


객체참조에 의한 전달 (파이썬-변경불가능객체전달)

파이썬의 경우는 C언어처럼 변수 내부의 값을 담고 있는 형태가 아니라, 메모리에 저장되어있는 값을 참조하는 형태. 변수 이름은 값을 가리키는 형태입니다!


객체참조에 의한 전달 (파이썬-변경가능객체전달)

람다함수

람다함수는 이름이 없는 함수입니다. return 문이 따로 없기 때문에 x=10과 같이 할당문은 쓰일 수가 없습니다. 보통은 자주 사용하지 않는 함수를 람다함수로 씁니다. 아래와 같이 쓰일 수 있습니다.
